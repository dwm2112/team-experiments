# Upwork Proposal Template

Use this template in Stage 2 (`/team:proposal`). Fill every `[PLACEHOLDER]`. Target **200–300 words**. Cut a story if needed; never pad.

---

## Paste-ready body

```
Hi — your post reads less like "[SURFACE_ASK]" and more like "[REAL_ASK]". That's the work I do best.

A few relevant examples:

1. [STORY_1_LABEL]: [STORY_1 — 2–3 sentences: problem, how you clarified, what you delivered]
2. [STORY_2_LABEL]: [STORY_2 — 2–3 sentences with outcome]
3. [STORY_3_LABEL]: [STORY_3 — 2–3 sentences with outcome]

On AI tools: I use Claude Code / Codex to explore unfamiliar APIs and verify edge cases faster — not to paste errors until something runs. I still own the architecture and the final review.

One honest note: [HONESTY_LINE — real gap + how you'd figure it out]

Two questions before I'd propose a stack or approach:
1. [QUESTION_1 — business-context]
2. [QUESTION_2 — stack / data / integration]

I'm [LOCATION], available [HOURS_PER_WEEK] hrs/week, and my rate is $[RATE]/hr[RATE_NOTE].

Happy to jump on a short call, or I can look at a specific pain point and outline how I'd approach it. Either works.

— [YOUR_NAME]
```

## Fill-in guide

| Placeholder | What to put |
|-------------|-------------|
| `[SURFACE_ASK]` | What the posting literally asks for (e.g. "build a web app") |
| `[REAL_ASK]` | The underlying need (e.g. "take a vague business problem, figure out requirements, and ship end-to-end") |
| `[STORY_N_LABEL]` | Short label: Vague ask → shipped / Automation / Data integration / etc. |
| `[STORY_N]` | Specific, true story with domain + decision + deliverable |
| `[HONESTY_LINE]` | One real gap + approach — never invent |
| `[QUESTION_1]` | Business-context question (why before how) |
| `[QUESTION_2]` | Stack / data / integration question about *their* system |
| `[LOCATION]` | Credible location (e.g. "US-based (Austin, TX)") |
| `[HOURS_PER_WEEK]` | Sustainable weekly hours |
| `[RATE]` | Hourly rate matching their range if stated |
| `[RATE_NOTE]` | Optional: " — within your $X–Y range" |
| `[YOUR_NAME]` | Name as on Upwork |

### Story priority

1. Prefer stories that match signals in the RFP (payments, recon, legacy stack, vague-ask delivery).
2. Prefer recent, measurable, business-facing stories over tech demos.
3. Two strong stories beat three vague ones — drop the weakest slot rather than invent fluff.

### Do

- Mirror their language.
- Lead with behavior and outcomes, not a skills laundry list.
- Ask 1–2 sharp questions (demonstrate collaboration *in* the proposal).
- One real honesty line.

### Don't

- Open with "I am a passionate … with X years…"
- Paste a resume or framework laundry list.
- Claim tools/APIs you don't have.
- Buzzwords, stacked emojis, or obvious AI tone.
- Invent gaps or stories.

### Checklist before send

- [ ] All placeholders replaced
- [ ] 2–3 stories specific and true
- [ ] Honesty line real
- [ ] Questions about *their* business/stack
- [ ] Location + hours + rate filled
- [ ] ~200–300 words
- [ ] Reads like a person out loud
